# Inspire CLI - AI Notes

> Quick reference for AI assistants working with the Inspire CLI codebase.

As of: 2026-03-21

## Project Overview

Inspire CLI is a command-line tool for the Inspire HPC platform:
- Submit and manage jobs on GPU clusters
- Monitor status and stream logs
- Sync repositories to remote Bridge runners
- Launch and SSH into notebook sessions
- Inspect GPU/node availability

## Public Sync Workflow

Treat `.gitignore` as the source of truth for private/internal boundaries.

Use `bin/sync-github-public.sh` for `github-public` syncs. It enforces an explicit allowlist and prune behavior so removed files in synced paths are deleted from `github-public` too.

Allowlist (sync from local `main` to `github-public/main`):
- `README.md`
- `CHANGELOG.md`
- `pyproject.toml`
- `uv.lock`
- `inspire/`
- `tests/`

Post-sync guards enforced by the script:
- Fail if stale files remain inside allowlist paths (example: `inspire/platform/web/browser_api/notebooks.py`).
- Fail if `inspire/` changed but `pyproject.toml` or `uv.lock` does not match local `main`.

Recommended command:

```bash
bin/sync-github-public.sh main github-public/main public-sync
```

Then review and push:

```bash
git diff --stat github-public/main..HEAD
git push github-public public-sync:main
```

## SSH/SCP Reliability Semantics

- `inspire bridge ssh` and `inspire bridge scp` should validate explicit `--bridge` names first and return `BridgeNotFound` guidance when missing (`inspire tunnel list`).
- For tunnel failures (profile exists but SSH unavailable), the recovery flow is `inspire tunnel status`, then re-create via `inspire notebook ssh <notebook-id> --save-as <name>` when needed.
- There is no `inspire tunnel start` command in this CLI.
- `BridgeProfile` supports optional `notebook_id`; notebook SSH fast-path reuse is only valid when cached `notebook_id` matches the requested notebook.
- `generate_ssh_config()` must reuse `_get_proxy_command()` so `ProxyCommand` remains shell-quoted and safe for URLs/paths with special characters.

## Sync Semantics (As of 2026-03-21)

- `inspire sync` defaults to `--transport ssh --source bundle`.
- Push is skipped by default (`--push-mode skip`); `--no-push` is an explicit alias.
- Push occurs only when `--push-mode required` or `--push-mode best-effort` is provided.
- `--source auto` is removed.
- `--branch` is removed.
- `--source remote` is only valid with `--transport ssh`.
- `--transport workflow` is explicit fallback mode only (no automatic SSH->workflow fallback).
- Bridge selection is deterministic, then availability-checked with `is_tunnel_available(...)`.
- For `--source bundle`, bridge priority is: offline CPU -> offline non-CPU -> internet CPU -> internet non-CPU.
- For `--source remote`, bridge priority is: internet CPU -> internet non-CPU -> offline CPU -> offline non-CPU.
- `default_bridge` is only a tie-breaker inside the same priority bucket.
- If `--source remote` selects a bridge with no internet, sync fails with a hint to use `--source bundle`.

## Configuration Model

### Precedence
```text
Defaults < Global TOML < Project TOML < Environment variables
```

### Config File Locations
- Global: `~/.config/inspire/config.toml`
- Project: `./.inspire/config.toml`

### High-Impact Env Vars
- `INSPIRE_USERNAME`
- `INSPIRE_PASSWORD`
- `INSPIRE_TARGET_DIR`
- `INSPIRE_BASE_URL`
- `INSPIRE_SHM_SIZE`
- `INSPIRE_BRIDGE_ACTION_TIMEOUT`
- `INSP_GITEA_REPO`
- `INSP_GITEA_TOKEN`
- `INSP_GITEA_SERVER`

## Code Patterns

### Load Config
```python
from inspire.config import Config

cfg = Config.from_env(require_credentials=True)
cfg, sources = Config.from_files_and_env(require_credentials=True)
```

### Get Authenticated API Client
```python
from inspire.cli.utils.auth import AuthManager

api = AuthManager.get_api(cfg)
```

### Click Command Pattern
```python
import click
from inspire.cli.context import Context, pass_context

@click.command()
@pass_context
def my_command(ctx: Context) -> None:
    if ctx.json_output:
        click.echo('{"ok": true}')
    else:
        click.echo("ok")
```

## Where To Edit

1. Add a config option:
   - `inspire/config/options/`
   - `inspire/config/schema.py` / `schema_models.py`
   - `inspire/config/models.py`
2. Add a command:
   - Command registry under `inspire/cli/commands/<group>/__init__.py`
   - Implementation in `inspire/cli/commands/<group>/...`
3. Add OpenAPI behavior:
   - `inspire/platform/openapi/`
4. Add browser-only notebook/resource behavior:
   - `inspire/platform/web/browser_api/`
5. Add tests:
   - `tests/test_*.py`

## Notes

- Keep reusable logic out of CLI-only `_impl` paths when possible.
- Prefer `inspire/config`, `inspire/platform`, and `inspire/bridge` for shared logic.
- For public syncs, double-check changes against `.gitignore` before pushing.
