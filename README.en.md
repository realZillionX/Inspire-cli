# Inspire CLI

CLI for the Inspire training platform — notebook lifecycle, distributed job / hpc job submission, SSH tunneling, image management, and code sync in one tool.

> **README scope:** this file only covers installation and one-time setup.
>
> **Full operational guide:** [Inspire Skill - Platform Operations Manual](https://fudan-nlp.feishu.cn/wiki/D2RXwnZcQiUQadkadJgcC1aEnLh)
>
> **Confidential network setup:** [Clash 7897 network setup](https://fudan-nlp.feishu.cn/wiki/NDvbw0TZPiiNT2k1DzmcMovHnoc)

## Installation

```bash
# Via SSH (recommended)
uv tool install git+ssh://git@github.com/realZillionX/Inspire-cli.git

# Or via HTTPS
uv tool install git+https://github.com/realZillionX/Inspire-cli.git
```

### Local Development

```bash
uv tool install -e .
inspire --help
```

## One-time Setup

| Step | Action | Notes |
| --- | --- | --- |
| 1. Network prerequisite | Follow the Feishu doc above | Only needed when this machine cannot reach `*.sii.edu.cn` directly |
| 2. Auto-discover | `inspire init --discover -u <username> --base-url https://qz.sii.edu.cn` | Writes global config to `~/.config/inspire/config.toml` and project config to `./.inspire/config.toml` |
| 3. Verify | `inspire config show` / `inspire config check` | Inspect merged config and validate authentication |

Notes:

- `INSPIRE_PASSWORD` can be exported temporarily or persisted in your shell startup file. If you want a long-lived per-account setup, you can also store it in `[accounts."<username>"].password` inside `~/.config/inspire/config.toml`.
- Default `discover` already tries to populate `workdir` and `shared_path_group`.
- Only use `--probe-shared-path` when some shared-path groups remain unknown; it is slower because it creates temporary `CPU notebooks`.

## Config Files

| Layer | Default path | Purpose |
| --- | --- | --- |
| Global config | `~/.config/inspire/config.toml` | Account, password, base URL, and a small amount of machine-level config |
| Project config | `./.inspire/config.toml` | Project, workspace, and compute-group metadata generated by `discover` |
| Environment variables | Current shell / CI environment | Temporary overrides and secret fallbacks |

Default precedence is `defaults < global TOML < project TOML < environment variables`. If project config sets `cli.prefer_source = "toml"`, project TOML wins on conflicts.

Minimal global config example:

```toml
[auth]
username = "your_username"

[accounts."your_username"]
password = "your_password"

[api]
base_url = "https://qz.sii.edu.cn"
```

Project-level defaults such as base images, remote paths, and shared-directory conventions should live in the project's own `AGENTS.md`, not in this README.

## Common Environment Variables

For the full list, run `inspire config env`.

| Variable | Description |
| --- | --- |
| `INSPIRE_USERNAME` | Platform username |
| `INSPIRE_PASSWORD` | Platform password fallback |
| `INSPIRE_BASE_URL` | API base URL |
| `INSPIRE_GLOBAL_CONFIG_PATH` | Override path for the global config file |

## Development & Contributing

If you only want to try the CLI locally, use the "Local Development" section above. If you are changing the repository itself, run at least the same checks as CI:

| Goal | Command |
| --- | --- |
| Install dev dependencies | `uv sync --group dev` |
| Run tests | `uv run pytest -x -q --tb=short` |
| Run lint | `uv run ruff check inspire tests` |
| Check formatting | `uv run black --check inspire tests` |

Optional preflight: `uv run pre-commit run --all-files`

Use [Conventional Commits](https://www.conventionalcommits.org/) prefixes such as `feat:`, `fix:`, `docs:`, and `chore:`. If a change affects user-visible behavior, update [CHANGELOG.md](CHANGELOG.md) in the same patch.
