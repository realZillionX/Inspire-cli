# Inspire CLI - AI Notes

> Quick reference for AI assistants working with the Inspire CLI codebase.

## Project Overview

Inspire CLI is a command-line tool for the Inspire HPC platform:
- Submit and manage jobs on GPU clusters
- Monitor status and stream logs
- Sync repositories to remote Bridge runners
- Launch and SSH into notebook sessions
- Inspect GPU/node availability

## Public Sync Scope (.gitignore)

Treat `.gitignore` as the source of truth for public-sync boundaries.

Ignored/internal paths currently include:
- `.inspire/`
- `internal/`
- `scripts/`
- `API_ENDPOINTS.md`
- `CLAUDE.md`
- `config.toml.example`
- `docs/rtunnel-ssh-setup.md`

When preparing `github-public` sync, avoid adding docs/tests/examples that depend on ignored files.

## SSH/SCP Reliability Semantics

- `inspire bridge ssh` and `inspire bridge scp` should validate explicit `--bridge` names first and return `BridgeNotFound` guidance when missing (`inspire tunnel list`).
- For tunnel failures (profile exists but SSH unavailable), the recovery flow is `inspire tunnel status`, then re-create via `inspire notebook ssh <notebook-id> --save-as <name>` when needed.
- There is no `inspire tunnel start` command in this CLI.
- `BridgeProfile` supports optional `notebook_id`; notebook SSH fast-path reuse is only valid when cached `notebook_id` matches the requested notebook.
- `generate_ssh_config()` must reuse `_get_proxy_command()` so `ProxyCommand` remains shell-quoted and safe for URLs/paths with special characters.

## Configuration Model

### Precedence
```text
Defaults < Global TOML < Project TOML < Environment variables
```

### Config File Locations
- Global: `~/.config/inspire/config.toml`
- Project: `./.inspire/config.toml`

## Code Patterns

### Load Config
```python
from inspire.config import Config

cfg = Config.from_env(require_credentials=True)
cfg, sources = Config.from_files_and_env(require_credentials=True)
```

### Get Authenticated API Client
```python
from inspire.cli.utils.auth import AuthManager

api = AuthManager.get_api(cfg)
```

### Click Command Pattern
```python
import click
from inspire.cli.context import Context, pass_context

@click.command()
@pass_context
def my_command(ctx: Context) -> None:
    if ctx.json_output:
        click.echo('{"ok": true}')
    else:
        click.echo("ok")
```

## Where To Edit

1. Add a config option:
   - `inspire/config/options/`
   - `inspire/config/schema.py` / `schema_models.py`
   - `inspire/config/models.py`
2. Add a command:
   - Command registry under `inspire/cli/commands/<group>/__init__.py`
   - Implementation in `inspire/cli/commands/<group>/...`
3. Add OpenAPI behavior:
   - `inspire/platform/openapi/`
4. Add browser-only notebook/resource behavior:
   - `inspire/platform/web/browser_api/`
5. Add tests:
   - `tests/test_*.py`

## Notes

- Keep reusable logic out of CLI-only `_impl` paths when possible.
- Prefer `inspire/config`, `inspire/platform`, and `inspire/bridge` for shared logic.
- For public syncs, double-check changes against `.gitignore` before pushing.
